---
title: "#[validators]"
description: "Procedural macro for rebuilding typed machines from persisted data."
---

The `#[validators]` macro is applied to an `impl PersistedType` block to generate typed machine rebuilds from stored rows.

## Usage

```rust
use statum::{machine, state, validators, Error};

#[state]
enum TaskState {
    Draft,
    InReview(String),
    Published,
}

#[machine]
struct Task<TaskState> {
    id: u64,
}

struct TaskRow {
    id: u64,
    status: &'static str,
    reviewer: Option<String>,
}

#[validators(Task)]
impl TaskRow {
    fn is_draft(&self) -> statum::Result<()> {
        if self.status == "draft" {
            Ok(())
        } else {
            Err(Error::InvalidState)
        }
    }

    fn is_in_review(&self) -> statum::Result<String> {
        if self.status == "in_review" {
            self.reviewer.clone().ok_or(Error::InvalidState)
        } else {
            Err(Error::InvalidState)
        }
    }

    fn is_published(&self) -> statum::Result<()> {
        if self.status == "published" {
            Ok(())
        } else {
            Err(Error::InvalidState)
        }
    }
}

fn main() -> statum::Result<()> {
    let row = TaskRow {
        id: 7,
        status: "draft",
        reviewer: None,
    };

    let row_id = row.id;
    let _task = row.into_machine().id(row_id).build()?;
    Ok(())
}
```

## Syntax

`#[validators(MachineName)]` takes the machine struct name as its only argument. Statum resolves the state family from the machine definition — you do not need to specify the state enum separately.

## Naming convention

Define one `is_{state}` method per state variant. The method name must match the snake_case form of the variant name:

| Variant | Method name |
|---|---|
| `Draft` | `is_draft` |
| `InReview` | `is_in_review` |
| `Published` | `is_published` |

## Return types

| State kind | Return type |
|---|---|
| Unit state (no data) | `statum::Result<()>` |
| Data-bearing state | `statum::Result<StateData>` |

Return `Ok(())` or `Ok(data)` when the row matches the state. Return `Err(statum::Error::InvalidState)` when it does not.

## Machine field bindings

Machine fields are available by name inside validator bodies through generated bindings. Persisted-row fields still live on `self`:

```rust
#[validators(Task)]
impl TaskRow {
    fn is_draft(&self) -> statum::Result<()> {
        // `self.status` is a TaskRow field
        // machine fields (e.g., `id`) are available as generated bindings
        if self.status == "draft" {
            Ok(())
        } else {
            Err(Error::InvalidState)
        }
    }
}
```

## Generated API

### `into_machine()`

Rebuilds a single row into a typed machine. Returns a builder that accepts machine fields before calling `.build()?`:

```rust
let machine = row.into_machine().id(row.id).build()?;
```

### `.into_machines()`

Rebuilds a collection of rows when all items share the same machine fields. Requires the `IntoMachinesExt` trait to be in scope for cross-module use:

```rust
use task::IntoMachinesExt as _;

let machines = rows.into_machines().id(shared_id).build()?;
```

### `.into_machines_by(|row| machine::Fields { ... })`

Rebuilds a collection of rows when each item needs different machine field values:

```rust
use task::IntoMachinesExt as _;

let machines = rows.into_machines_by(|row| task::Fields { id: row.id }).build()?;
```

## Matching rebuilt output

The result of `.build()?` is a `machine::SomeState` enum. Match on its variants to access the typed machine:

```rust
match machine {
    task::SomeState::Draft(t) => { /* t: Task<Draft> */ }
    task::SomeState::InReview(t) => { /* t: Task<InReview>, t.state_data: String */ }
    task::SomeState::Published(t) => { /* t: Task<Published> */ }
}
```

## Async support

If any validator method is `async`, the generated `into_machine()`, `.into_machines()`, and `.into_machines_by(...)` builders become `async` as well. Await the `.build()` call:

```rust
let machine = row.into_machine().id(row.id).build().await?;
```

## Complete example

```rust
use statum::{machine, state, validators, Error};

#[state]
enum TaskState {
    Draft,
    InReview(String),
    Published,
}

#[machine]
struct Task<TaskState> {
    id: u64,
}

struct TaskRow {
    id: u64,
    status: &'static str,
    reviewer: Option<String>,
}

#[validators(Task)]
impl TaskRow {
    fn is_draft(&self) -> statum::Result<()> {
        if self.status == "draft" {
            Ok(())
        } else {
            Err(Error::InvalidState)
        }
    }

    fn is_in_review(&self) -> statum::Result<String> {
        if self.status == "in_review" {
            self.reviewer.clone().ok_or(Error::InvalidState)
        } else {
            Err(Error::InvalidState)
        }
    }

    fn is_published(&self) -> statum::Result<()> {
        if self.status == "published" {
            Ok(())
        } else {
            Err(Error::InvalidState)
        }
    }
}

fn main() -> statum::Result<()> {
    let row = TaskRow {
        id: 7,
        status: "in_review",
        reviewer: Some("alice".to_owned()),
    };

    let row_id = row.id;
    let machine = row.into_machine().id(row_id).build()?;
    match machine {
        task::SomeState::InReview(task) => assert_eq!(task.state_data, "alice"),
        _ => panic!("expected in-review task"),
    }
    Ok(())
}
```
